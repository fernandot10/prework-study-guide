Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • The style element defines the CSS styles associated with a website
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indidcates how much space we want around the content inside an element.
  • Linking an external style sheet: link rel="stylesheet" href="file/location/style.css"

Git

  • GitHub issues gives developers and teams an easy way to keep track of key tasks and progress
  • Markdown syntax is used to write descriptions for issues
  • Default branch is called Main
  • A copy of the main branch is called a feature branch
  • A pull request (PR) is used to merge a comparison branch (feature branch)
  • Current branch status: git status
  • Creating a new feature branch in Gitbash: git checkout -b feature/'branch-name'
  • Add starter code to current branch: git add -A
  • Commiting branch: git commit -m 'description'
  • Local branch in sync? git pull origin main
  • Push changes to remote branch: git push origin 'feature/branch-name'
  • Switching between branches: git checkout 'branch-name'
